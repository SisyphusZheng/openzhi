# v0.16 Roadmap — WC Package Protocol

> Patch-level detail for Phase 2 of the [global roadmap](./ROADMAP.md).

## Overview

v0.16 turns the 3-field `PackageIslandMeta` into a CEM-compatible package
protocol. After v0.16.0, LessJS can describe WC packages as data instead of
only executing package code.

**Key rule**: v0.16.0 ships the protocol even if v0.16.1 integration is not
ready. Do not let build-time SSR-aware package integration block the manifest
and registry foundation.

## v0.16.0 — Manifest + Registry

**SOP**: `docs/sop/v0.16.0-package-protocol.md`
**ADR**: ADR-0026 (partial — Phase 4 deferred)
**Est. sessions**: 3-4

### Scope

| # | Item                        | Files                                         | Est.           |
| - | --------------------------- | --------------------------------------------- | -------------- |
| 1 | Manifest type system        | `core/src/types.ts`, `core/src/index.ts`      | 1 session      |
| 2 | Backward compat adapter     | `core/src/registry.ts` or `core/src/types.ts` | 0.5 session    |
| 3 | @lessjs/ui manifest         | `ui/src/manifest.ts`, `ui/src/index.ts`       | 0.5 session    |
| 4 | Local registry + validation | `core/src/registry.ts`                        | 1-1.5 sessions |

### Scope 1: Manifest type system

Add to `packages/core/src/types.ts`:

- `LessPackageManifest` — top-level package descriptor (20+ fields)
- `LessModule` — module entry within a package
- `LessDeclaration` — custom element declaration
- `LessElementExtensions` — SSR/DSD/hydration declarations
- `LessPackageExtensions` — package-level LessJS config
- CEM-compatible subtypes: attributes, members, events, slots, CSS properties, CSS parts
- `LessExport` — named export descriptor

### Scope 2: Backward compat adapter

```ts
packageIslandFromManifest(manifest: LessPackageManifest): PackageIslandMeta[]
```

- Preserve existing `@lessjs/ui` island behavior
- Mark `PackageIslandMeta` as `@deprecated` (removal target: v0.18+)

### Scope 3: @lessjs/ui manifest

- `packages/ui/src/manifest.ts` — export `manifest: LessPackageManifest`
- 7 components (less-button, less-card, less-input, less-code-block,
  less-layout, less-theme-toggle, less-hero-ping, less-dialog) with CEM metadata
- `islands` array generated via `packageIslandFromManifest(manifest)`

### Scope 4: Local registry + validation

New file `packages/core/src/registry.ts`:

```ts
register(manifest: LessPackageManifest): void
getByTagName(tagName: string): LessDeclaration | undefined
getAll(): LessPackageManifest[]
validate(manifest: LessPackageManifest): ValidationResult
generateIndex(): RegistryIndex
```

Validation rejects:

1. Missing required fields
2. Invalid custom element names
3. Unsafe module paths
4. Invalid layer or hydrate strategy
5. Duplicate tag names inside a package
6. Unresolved export declaration references
7. Registry-level duplicate tag names (unless explicitly allowed)

## v0.16.1 — Build-Time Package Integration (Conditional)

**SOP**: `docs/sop/v0.16.1-build-time-package-integration.md`
**Est. sessions**: 3-4
**Risk**: HIGH — may defer to v0.17 if integration is unstable

### Scope

| # | Item                       | Files                                                      | Est.         |
| - | -------------------------- | ---------------------------------------------------------- | ------------ |
| 1 | Adapter reads manifests    | `route-scanner.ts`, `build-client.ts`, `entry-renderer.ts` | 1 session    |
| 2 | SSR-aware island rendering | render pipeline changes                                    | 1.5 sessions |
| 3 | Client entry generation    | CEM metadata for events/slots/attrs                        | 0.5 session  |
| 4 | Build report integration   | extend `dsd-report.json`                                   | 0.5 session  |

### SSR-aware rendering rules

| Manifest flags          | Behavior                                         |
| ----------------------- | ------------------------------------------------ |
| `ssr: true, dsd: true`  | Include in SSR bundle, render with `renderDSD()` |
| `ssr: true, dsd: false` | Include in SSR bundle, use pure-island path      |
| `ssr: false`            | Skip SSR, client bundle only                     |

### Exit criteria

- v0.16.1 proves that the package protocol can drive real rendering behavior.
- If integration is unstable, move to v0.17 instead of blocking v0.16.0.

### Verification

- [ ] Package island with `ssr: true` appears in SSG output
- [ ] Package island with `ssr: false` appears only in client output
- [ ] Invalid manifest fails before generated code is emitted
- [ ] Local islands are unaffected
- [ ] `deno task test`, `deno task build`, `deno task test:e2e`
