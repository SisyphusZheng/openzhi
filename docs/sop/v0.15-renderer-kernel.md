## Phase 3: Code — Upgrade to RendererProtocol

Goal: Implement the interfaces from ADR-0025.

### 3.1 Add Protocol types to `core/types.ts`

- `RendererProtocol` (superset of `RenderAdapter`, adds `name: string`)
- `RenderOutput` (html + errors + metrics + hydrationHints)
- `RenderInput` (tagName + componentClass + props + dsdOptions + nestingDepth)
- `RenderError` (phase + tagName + message + recoverable)
- `HydrationHint` (tagName + layer + events + strategy)
- Keep `RenderAdapter` as deprecated alias for `RendererProtocol`

### 3.2 Update `adapter-registry.ts`

- `registerAdapter()` accepts `RendererProtocol`
- Add `getRegisteredAdapters(): readonly RendererProtocol[]`
- Keep backward compat: `getAdapter()` returns last registered

### 3.3 Change `renderDSD()` return type

- `renderDSD()` → returns `Promise<RenderOutput>`
- Add `renderDSDString()` → returns `Promise<string>` (backward compat wrapper)
- Collect errors, metrics, hydrationHints into `RenderOutput`
- Remove `DsdRenderCollector` parameter from `renderDSD()` — metrics are now
  part of `RenderOutput`

### 3.4 Update `render-nested.ts`

- Propagate `RenderOutput` errors up to parent
- Pass `nestingDepth` through correctly

### 3.5 Update `adapter-lit/ssr.ts`

- Add `name: 'lit'` to registered adapter
- No other changes needed (isTemplate/render/extractStyles unchanged)

### 3.6 Update consumers

- `adapter-vite/src/cli/ssg-render.ts` — use `renderDSDString()` for HTML output
- `adapter-vite/src/entry-renderer.ts` — use `renderDSDString()`
- Any other callers of `renderDSD()` — switch to `renderDSDString()` or
  `renderDSD()` depending on whether they need structured output
