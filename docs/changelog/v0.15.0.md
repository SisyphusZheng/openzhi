# v0.15.0 — 2026-05-16

## Summary

Renderer Kernel Protocol — upgrade RenderAdapter to RendererProtocol with
structured output, error taxonomy, DSD build reports, and renderDSD module split.

## Changes

- **Protocol**: Replace `RenderAdapter` with `RendererProtocol` (adds `name`,
  structured `RenderOutput` with `html` + `errors` + `metrics` +
  `hydrationHints`)
- **Types**: Add `RenderOutput`, `RenderInput`, `RenderError`, `HydrationHint`
  to `@lessjs/core`
- **Backward compat**: Add `renderDSDString()` wrapper; deprecate bare
  `renderDSD()` string return
- **Module split**: `render-dsd.ts` (360 lines) → `render-dsd.ts` +
  `render-errors.ts` + `render-instantiate.ts` + `render-serialize.ts`
- **DSD report**: SSG builds emit `dsd-report.json` with aggregate render metrics
- **Adapter registry**: Support named adapters via `getRegisteredAdapters()`
- **create CLI**: Fix JSR remote version resolution; update scaffold to v0.15.0
- **ADR-0025**: Renderer Protocol technical specification

## Known Issues

- 3 JSR `unanalyzable-dynamic-import` warnings in adapter-vite (unchanged from
  v0.14.11)

## Breaking Changes

- `renderDSD()` now returns `Promise<RenderOutput>` instead of `Promise<string>`.
  Use `renderDSDString()` for backward-compatible string output.
- `RenderAdapter` is deprecated; use `RendererProtocol` (add `name: string`).
