# v0.17.3 — Multi-Framework Adapters

**Release date**: 2026-05-16

## Summary

Adds two new SSR adapter packages expanding LessJS beyond Lit:
`@lessjs/adapter-vanilla` for plain Web Components and
`@lessjs/adapter-react` for React-to-Web-Component bridging.

## Changes

- **@lessjs/adapter-vanilla**: New package for plain/vanilla Web Components
  whose `render()` returns a string directly.
  - `installVanillaAdapter()`: registers `'vanilla'` adapter for DSD rendering
  - `extractVanillaStyles()`: extracts static `styles` property (string or
    string array) from vanilla component classes
  - `DsdVanillaElement`: pre-composed base class extending `HTMLElement` with
    DSD detection + hydration event binding + AbortController cleanup
  - `WithDsdHydration()`: mixin for custom base class composition
  - Zero framework dependencies — pure DOM API

- **@lessjs/adapter-react**: New package for React components wrapped as
  Web Components.
  - `installReactAdapter()`: registers `'react'` adapter for DSD rendering
  - `isReactElement()`: detects React elements via `$$typeof` symbol
  - `renderReactToString()`: converts React elements to HTML strings via
    `ReactDOMServer.renderToStaticMarkup()` (lazy-loaded)
  - `DsdReactElement`: pre-composed base class with `getReactElement()`
    method for SSR rendering + client-side `createRoot()` mounting
  - `WithDsdHydration()`: mixin for custom base class composition
  - Peer deps: `react@^19`, `react-dom@^19`

- **Workspace config**: Both packages added to root `deno.json` workspace,
  imports, typecheck, and publish tasks.

- **Tests**: 26 new tests (9 adapter-vanilla, 8 adapter-react, plus
  mixin tests). Total: 554 unit tests + 92 e2e passing.

## Breaking Changes

None. This is a minor release — two new packages, no changes to existing
packages.

## Known Issues

- 3 JSR `unanalyzable-dynamic-import` warnings in adapter-vite (expected,
  runtime-only deps, not blocking publish)
- `DsdVanillaElement` and `DsdReactElement` are Proxy-based lazy
  initializers to avoid `ReferenceError` in non-DOM Deno test
  environments. In browser/DOM contexts they work as transparent
  class references.
