# LessJS 官网设计审查报告

> 审查范围：www 首页视觉设计、排版、色彩系统、一致性  
> 审查日期：2026-05-12

---

## 现状总结

网站整体可用，功能区完备，但在**视觉层次、色彩一致性、排版细节**上有明显提升空间。不差，但不够好。

---

## 具体问题

### 1. 色彩系统不统一

首页使用了多个相互独立的颜色体系：
- Hero / 代码条 / CTA：黑色 `#09090b`
- Feature 卡片图标：紫色 `#EEEDFE`、蓝色 `#E6F1FB`、绿色 `#E1F5EE`
- Use cases 区：无颜色，纯边框
- Benchmark 表：绿色胜、红色败
- Bundle 对比条：绿色、红色
- Demo 卡：`#18181b` 暗色

每个区块都有自己的颜色逻辑，缺少一个**贯穿全页的品牌色**。目前的品牌色只有一个——LessJS logo 的黑色。

**建议**：选一个点缀色作为"交互/亮点"标识（比如紫色 `#534AB7`），在 CTA 按钮、链接高亮、重要数据上统一使用。降低黑色块的使用频率——三个全宽黑色块（hero + 代码条 + CTA）有点多。

### 2. 首页八区块，节奏感弱

当前区块顺序：
Hero → 代码对比 → 3 卡片 → 4 用例 → 对比表 → 架构图 → Demo → 对比条 → 快速开始 → CTA

一共 **10 个区块**，每个宽度一样、间距一样、没有主次。用户滚动时感觉不到"现在看到哪了"。

**建议**：减少到 6-7 个区块，用不同的视觉重量区分主次：
- Hero（最重，全宽黑底）
- 核心卖点 3 卡片（中等）
- 代码对比（中重，全宽黑底）
- 对比表（轻，白底）
- Live Demo + Bundle 条（中，暗底）
- CTA（中重，全宽灰底/黑底）

### 3. 间距偏大，内容密度低

当前每两个区块之间都有 `2.5rem` 的 padding，加上区块自己的 padding，页面滚动距离很长但信息密度低。

**建议**：缩减到 `1.5rem-2rem`，区块内的间距也对应收紧。代码对比和架构图可以更紧凑。

### 4. Benchmark 表缺少视觉处理

当前就是一张 HTML 表格，没有任何视觉增强。作为"LessJS vs Next.js vs Astro"的对比，它的重要性值得更好的视觉呈现。

**建议**：在 "LessJS" 列用紫色背景高亮，或者在表格上方加一行标签"我们 vs 它们"。LessJS 列的文字加粗。

### 5. 对比条不够显眼

Bundle size 对比条在页面较下方。LessJS 的绿条只有 2% 宽度（0KB），Next.js 的红色满宽。这个视觉冲击力很大，但位置太靠下了。

**建议**：移到更靠前的位置，比如跟在代码对比之后。或者把宽度差距做得更夸张（动画从 0 增长到实际宽度）。

### 6. 字体统一但无个性

全站使用系统字体栈，安全但不突出。对于"Web 框架"的官网来说，字体选择是品牌的一部分。

**建议**（低优先级）：hero 标题可以考虑用更粗的 `font-weight: 600` 甚至 700，和正文（`font-weight: 400`）拉开差距。目前 hero 标题是 500，和副标题差别太小。

### 7. 差距分析

与同类框架官网对比（Astro、Next.js、Vite）：

| 维度 | LessJS | Astro | Next.js |
|------|--------|-------|---------|
| Hero 冲击力 | 中等（终端加分） | 强（视频/动画） | 强（大标题+CTA） |
| 色彩一致性 | 弱 | 强（紫色品牌色贯穿） | 强（黑色+蓝色） |
| 区块节奏 | 平 | 有起伏 | 有起伏 |
| 内容密度 | 低 | 中 | 中高 |
| 交互元素 | ✅ terminal | — | — |

## 优先修复建议

**高（一小时内可改）：**
1. 选定一个品牌点缀色，加到关键元素上（CTA 按钮 hover 色、链接色）
2. 精简区块数，合并或删除冗余区块
3. LessJS 列在 benchmark 表中加视觉高亮

**中（半天）：**
4. 调整全局间距，提高内容密度
5. 给 benchmark 表做更好的视觉设计

**低（后续版本）：**
6. Hero 标题字重调整
7. 品牌字体引入
